## What Was Built

### 1. Edition Locking — Inventory Grid

When a set has `has_unlimited = false` in the `sets` table, the unlimited qty inputs in the inventory grid are greyed out and non-editable.

**db.js — `getCardsBySet()`:**
- Now fetches `has_unlimited` and `has_first_ed` from `sets` table for the searched set code
- Both flags propagate onto every card object returned

**inventory.js:**
- `buildCardRow()` stores `data-has-unlimited` on the `<tr>` element
- Passes `disabled = !card.has_unlimited` to `un_nm/lp/mp` and `binder_un_nm` cells
- `doSave()` reads `tr.dataset.hasUnlimited` and forces all unlimited fields to `0` before saving if 1st-ed-only
- `_editionShownSets` Set added at module level (tracks per-session toast state)
- Staff reminder toast fires once per set per session when `has_unlimited && has_first_ed`: "Check the bottom-left corner for a 1st Edition stamp. No stamp = Unlimited."

### 4. Bulk Price Updater — Full Rewrite

`bulk-price.html` rebuilt from scratch as a **standalone module** — no dependency on `js/config.js` or `js/db.js`.

| | Old | New |
|---|---|---|
| Data source | TCGCSV (2 endpoints) | YGOPRODeck `tcgplayer_data=true` (1 call) |
| Match key | card_number\|rarity → card_number → card_name | card_number\|rarity only |
| Fields written | tcg_market_price, tcg_low_price, api_id, 6 listing price cols, HR eBay price | `tcg_price` + `tcg_low_price` + `tcg_price_cad` |
| eBay HR lookup | Yes (Netlify function) | Removed |
| Listing price multipliers | Yes | Removed |
| api_id backfill | Yes | Removed |
| External JS deps | js/config.js, js/db.js | None |
| Options | 4 checkboxes | 1 checkbox (overwrite) |

**USD → CAD conversion:**
- Fetches live rate from `https://api.frankfurter.app/latest?from=USD&to=CAD`
- Displays rate used in the stats row
- Writes `tcg_price_cad = tcg_price × rate` alongside every USD price update
- `tcg_price_cad numeric` column added to `card_inventory` (SQL run manually in Supabase)
